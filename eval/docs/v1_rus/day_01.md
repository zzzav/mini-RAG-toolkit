# День 1 — Старт проекта Python

## Что сделали
- Создали виртуальное окружение (venv) и настроили установку зависимостей через pip.
- Подключили инструменты качества:
  - pytest для тестирования
  - black для форматирования
  - ruff для линтинга
- Сформировали базовую структуру проекта (src/, tests/).
- Зафиксировали правила работы:
  - коммит в git после завершения дня
  - единый стиль кода (black) и проверки (ruff)
  - все новые функции покрываем тестами.

## Результат
Есть “скелет” репозитория, в котором удобно и безопасно развивать CLI‑утилиты и модульную логику под будущий RAG.
