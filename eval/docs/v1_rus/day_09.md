# День 9 — Query normalization + synonyms

## Что сделали
- Вынесли единую нормализацию запроса:
  - lower
  - пунктуация → пробел
  - split
  - stop_words
- Добавили расширение синонимами:
  - `expand_tokens`
  - контроль max_expansions
  - дедупликация и сохранение порядка
- Интегрировали в:
  - `vector_search.search`
  - `tfidf_search` (запросная сторона)
- Добавили флаги CLI:
  - `--use-synonyms`
  - `--no-stop-words`

## Тесты
- Нормализация (пунктуация/пробелы/stop_words).
- Синонимы (порядок/лимит/дедуп).
- Интеграция: “bill → invoice” улучшает top‑1.

## Результат
Retrieval стал устойчивее к формулировкам пользователя и “вариантам слов”.
