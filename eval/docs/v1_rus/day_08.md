# День 8 — Evaluation-lite: Recall@k и MRR@k

## Что сделали
- Реализовали модуль оценки retrieval:
  - `EvalCase(query, relevant)`
  - загрузка JSONL датасета
  - метрики:
    - Recall@k (binary: найден ли хотя бы один релевантный)
    - MRR@k (1/rank первого релевантного)
  - `evaluate()` → `EvalReport` (mean метрики + per-case)

## CLI
- Добавили CLI для прогонов eval (text/json, out).

## Тесты
- Unit на математику recall/mrr.
- Интеграционный тест на реальном индексе.
- Добавили “плохой” кейс, чтобы тест ловил деградации.

## Результат
Появилась объективная проверка retrieval-качества, пригодная для регрессии.
