#курс_агенты #прикладнойn8n

❓ ПРО ЧТО

Это финальный, 7-й урок стартового курса ShipAI.

Здесь мы берем результат 6-го урока, где ИИ-разбор еще жил внутри основного workflow, и доводим ее до первой версии маленькой мультиагентной ИИ-системы.

Главный смысл урока:
ИИ-разбор перестает быть частью общего workflow и становится отдельной ролью внутри системы.

💡 ЧТО ВНУТРИ

— доработка основного workflow из урока 6
— сохранение основного контура для приема текста и голоса
— вынос ИИ-разбора в отдельный workflow агента
— запуск агента по id задачи из шины
— чтение prompt_system и prompt_user из БД
— чтение llm и temperature из настроек по задаче
— отдельный TTL-workflow для автоматического завершения контекста
— запуск одного и того же агента из двух сценариев:
  1) по кнопке
  2) по TTL

❕ ПОЧЕМУ ЭТО ВАЖНО

1️⃣ Основной контур больше не делает чужую работу

Теперь основной workflow отвечает за прием входа, накопление контекста и взаимодействие с пользователем.

2️⃣ ИИ-разбор стал переиспользуемым

Один и тот же агент вызывается и вручную, и автоматически.

3️⃣ Поведение агента начинает задаваться данными БД

Промпты и настройки модели больше не захардкожены в одном месте внутри общей схемы, теперь они грузятся из БД.

4️⃣ В системе появляется нормальный жизненный цикл задачи

Минимально используем статусы:
new → ready_for_parse → done

5️⃣ Появляется первый настоящий слой ролей

Отдельно:
— взаимодействие с пользователем
— ИИ-агент
— TTL

То есть это уже не просто workflow с ИИ-узлом, а первая версия системы ролей.

🔍 ОБРАТИ ВНИМАНИЕ

— как изменился finish_context по сравнению с уроком 6
— где именно теперь происходит перевод задачи в ready_for_parse
— как агент читает задачу только в нужном статусе
— как собираются prompt_system и prompt_user
— где подгружаются llm и temperature
— как TTL проверяет, жив ли еще нужный context_id
— почему TTL не разбирает текст сам, а только передает задачу агенту

Ключевая мысль урока:
мы больше не строим один workflow, который делает всё подряд.

Мы начинаем строить систему, где каждая часть отвечает за свое:
вход,
память,
разбор,
контроль,
результат.

🎓 ЧТО ПОЛУЧИЛОСЬ В ИТОГЕ

После урока у нас есть первая версия ИИ-секретаря как первого ИИ-агента внутри минимальной мультиагентной схемы.

Система уже умеет:
— принимать текст и голос
— копить контекст в Redis
— хранить внутреннюю задачу в шине
— передавать разбор отдельному агенту
— автоматически завершать ожидание по TTL
— сохранять результат во внутреннюю задачу
— добавлять найденные задачи в Google Sheets
